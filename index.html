Web authentication using SSH keys and Go

This form takes the same format as your OpenSSH authorized_keys file
and the *.pub file OpenSSH generates for a keypair. (see OpenSSH sshd(8) manual page)
It supports SSH-RSA, SSH-DSA, ECDSA-SHA2-NISTP{256,384,521}.
The server itself currently offers a RSA key, length 4096 bit and fingerprint
SHA256:IW0JCfRu0QMiR5ffaQQmnEGzXMe3lgtq524wahMFXo8.

Inspired by GitHub @altitude Demo, Source.

Privacy note This demo server only keeps a normal request log containing
your IP address and the executed action (HTTP Header contents). Your public key is only
logged in case of an error (in order to make an effort to improve usability) and otherwise
kept in memory for a maximum of 10 minutes.

A normal application would store the public key chosen along with the normal user credentials.
